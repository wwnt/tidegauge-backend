# Datalogger 硬件需求规格书

文档版本：V1.4  
状态：草案  
更新日期：2026-03-24

## 1. 目的

本文档用于定义 datalogger 硬件的最小接口需求，作为硬件设计和验收依据。

## 2. 系统要求

- 可稳定运行 Linux
- 采用耐久型存储，建议使用 `eMMC`
- 应具备耐腐蚀能力，适应海边环境
- 设计寿命应尽量长

## 3. 接口需求

整机必须至少具备以下接口或接入能力：

- `1` 路以太网接口
- `2` 路 `RS485` 接入能力
- `1` 路 `SDI-12` 接入能力
- `1` 路数字输入
- `1` 路 `0-5V` 电压模拟量输入
- `1` 路 `12V` 直流电源输入
- `USB Host` 接入能力

说明：

- `RS485` 可采用板载接口或外接转换模块实现
- `SDI-12` 可采用板载接口或外接转换模块实现
- 数字输入应可作为通用 `GPIO` 使用
- 模拟量输入范围为 `0-5V`
- 模拟量输入可为板载接口，也可通过外部采集模块实现
- 若通过外接 `USB` 转串口/转换模块实现 `RS485`、`SDI-12` 等接入能力，至少需要 `3` 个 `USB Host`

## 4. 验收要求

- 能稳定运行 Linux 系统
- 以太网接口工作正常
- `2` 路 `RS485` 可正常工作
- `1` 路 `SDI-12` 可正常工作
- 数字输入可正常识别
- 模拟量输入可正常识别
- `USB Host` 可正常识别外接设备
